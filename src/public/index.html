simple network monitor · v0.5.1

// demo mode — public hosts only (LAN IPs like 192.168.x or 10.x can't be reached from this cloud sandbox). Data resets on redeploys. SSH checks are disabled. To monitor your own network, self-host via Docker Hub or git clone.

ICMP = ping · TCP = connect to port (works through VPNs blocking ICMP) · SSH = connect and collect CPU / RAM / disk metrics (your ~/.ssh key must be authorized on the target host) · services = comma-separated systemd unit names to monitor in the details panel
